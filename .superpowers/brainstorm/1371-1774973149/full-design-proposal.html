Combat Positioning & Movement — Full Design Proposal

Relationship-based engagements, organic movement, and the facing fix

Part 1: Facing Fix

Current (Bug)
• SetPlannedTarget() called by attackers overwrites defender's look target
• CombatAILogic.UpdateFlip() fights with CharacterVisual.LateUpdate()
• Result: flip-flop every frame
Proposed Fix
• Only a character's own AI can set its look target
• SetLookTarget called in SetActionIntent only applies to the acting character
• Remove redundant UpdateFlip from CombatAILogic (let CharacterVisual.LateUpdate handle it)
• Single authority: CharacterVisual._lookTarget, set only by own combat AI

Part 3: Movement Behaviors

⚔️ Engagement Forms
• Both characters walk toward each other (meet in middle)
• Melee closes most of the gap
• Ranged takes a few steps then stops at preferred distance
🚶 Idle (Waiting for Turn)
• Subtle random sway around position (~0.5-1m drift)
• If 2:1+ numbers advantage: melee circles to surround
• Ranged stays more planted
💥 Attacking (Turn Active)
• Melee: moves into range, strikes, backs off to create small gap (dynamic spacing, no fixed "home")
• Ranged: shoots from current position, does NOT flee when approached
⏸️ Unengaged
• Barely moves — no target, no engagement
• Waiting for mutual targeting to form an engagement
• If has a one-way target: walks toward them to pressure a response
💀 Target Dies
• Immediately auto-acquire new target
• Join existing engagement if new target is in one
• Form new engagement if mutual targeting established
🏟️ Battle Zone
• Soft anchor at fight origin point
• Characters can drift but are gently pulled back if too far
• Fight stays roughly where it started

Part 5: What Gets Reworked

❌ Removed / Replaced
• CombatFormation.cs — fixed ring slots
• EngagementGroup.cs — replaced by targeting graph
• Proximity-based engagement creation in Coordinator
• Deterministic slot assignment by GetInstanceID()
• UpdateFlip() in CombatAILogic (redundant)
✅ New / Reworked
• Targeting graph — tracks who targets whom, detects mutuals
• Engagement builder — forms/splits/merges from targeting graph
• Organic positioner — dynamic spacing, no fixed slots
• Idle behaviors — sway + circling based on numbers
• Facing authority — single-owner look target control
• Approach system — meet-in-middle with range awareness
