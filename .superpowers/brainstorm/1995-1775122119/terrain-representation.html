Terrain Representation: Which approach?

How terrain types are physically laid out on maps — considering weather transforms, plant growth, and footstep detection

A

Zone-Based (Painted Regions)

Large collider areas define terrain type. Weather transforms entire zones at once.

Pros

  • Simple — few objects per map
  • Easy to author in the editor
  • Low performance cost
  • Matches your existing ZoneType system

Cons

  • No granularity — rain turns ALL dirt to mud at once
  • Can't have a small puddle next to dry ground
  • Plant growth limited to "zone has plants" — no specific positions
  • Weather transitions feel artificial (instant flip)
B

Hybrid: Zone Base + Cell Overlay

Zones define the base terrain (what this area naturally is). A lightweight cell grid tracks current state per tile — weather gradually transforms cells within a zone. Wind direction affects which cells transform first.

Pros

  • Gradual weather transitions — mud spreads from low points, snow accumulates
  • Wind pushes snow/rain directionally across cells
  • Temperature thresholds per-cell: snow melts in sun-exposed cells first
  • Plant growth at specific cell positions (fertile cells)
  • Footstep detection is a simple cell lookup
  • Offline simulation: MacroSimulator can math-forward cell states

Cons

  • More memory per map (cell grid)
  • More complex to implement
  • Need to define cell resolution (how big is each cell?)

Recommended — Best fit for wind/temperature atmosphere. Zone = "what this ground naturally is", Cell = "what it currently is right now". Weather gradually transforms cells, not instant zone flips.

C

Full Tile Grid (No Zones)

Every cell is independently authored. No zone concept — pure per-tile terrain.

Pros

  • Maximum granularity
  • Each tile fully independent

Cons

  • Huge authoring burden — every map cell must be painted
  • No concept of "natural state" — after weather passes, what does it revert to?
  • Doesn't leverage your existing zone system at all
  • Overkill for most maps

How the Hybrid (B) works with your atmosphere

Example: Rain event on a Dirt zone
Atmosphere State: Temperature=12C, Wind=(NE, 8 m/s), Precipitation=Rain
Zone Base Type: Dirt
Cell Transform Rule: Dirt + Rain + Saturation > 0.7 → Mud
t=0min  DirtDirtDirtDirtDirt
t=5min  DirtMudDirtDirtDirt ← low elevation saturates first
t=15min MudMudMudDirtDirt ← wind pushes rain NE→SW
t=30min MudMudMudMudMud ← fully saturated
Rain stops → Drying: Temperature + Wind speed determine evaporation rate. Cells revert to zone base type (Dirt) as saturation drops below threshold.
