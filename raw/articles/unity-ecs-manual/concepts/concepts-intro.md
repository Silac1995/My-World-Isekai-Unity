# Entity Component System Concepts

## Overview

The Entities package implements an entity component system (ECS) architecture for organizing code and data. According to the documentation, "An **entity** is a unique identifier, like a lightweight unmanaged alternative to a GameObject."

Key distinctions in this architecture:
- Entities serve as IDs linked to individual **components** containing entity data
- Unlike GameObjects, entities contain no code—they function as data units
- Systems process these data units
